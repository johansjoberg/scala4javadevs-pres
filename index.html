For comprehensions

Example

val names = List("Abraham", "Anna", "Ella", "Tomas", "Stellan")

//Declarative
val namesLongerThanFourChars = for(
  name <- names
  if name.length > 4
) yield name

println(namesLongerThanFourChars.mkString(", "))

//Imperative
var namesLongerThanFourChars = List[String]()
for(name <- names){
  if(name.length > 4) {
    namesLongerThanFourChars = name :: namesLongerThanFourChars
  }
}

println(namesLongerThanFourChars.mkString(", "  ))

Coding style

Declarative vs Imperative

Imperative example

case class Client(name: String, accounts:List[Int])

val clients = List(
  Client("Eric", List(1000, 400, 50)),
  Client("Michel", List(-10, 10, 0)),
  Client("Roger", List(0, 0, 0))
)
var clientsWithZeroBalance = List[Client]()
for(client <- clients){
  var sum = 0
  for(account <- client.accounts) {
    sum += account
  }
  if(sum == 0){
    clientsWithZeroBalance = client :: clientsWithZeroBalance
  }
}

println(clientsWithZeroBalance.mkString("\n"))

Object

Basics

object keyword used for defining a singleton

The apply method can be called directly though the object name

object is called a companion object if class with same name exists

The apply method is used as a factory in companion objects

Use object where you would use Java static

Object

Basics

object Name { //Companion object
  def apply(val firstName:String = "John", val lastName: String = "Doe") = {
    new Name(firstName, lastName)
  }

  def nameChild(parentName: Name, childFirstName: String) = {
    Name(childFirstName, parentName.lastName)
  }
}

val johnDoe = Name() //Same as calling Name.apply()
val janeDoe = Name.nameChild("Jane", johnDoe)
val robSmith = Name("Rob", "Smith")
val annSmith = Name(lastName = "Smith", firstName = "Ann")

Traits

Basics

trait keyword used for defining a trait

Like a Java interface, but method implementations

extends and with are used when mixing in a trait into a class

Self-types give access to the class that has the trait

Self-types restricts usage of the trait

Traits can be mixed in during object creation

Traits

Basics

class Person(val firstName: String, val lastName:String)

trait Speaker {
  def speak(sentences: String) = println(sentences)
}

trait Greeter { person: Person with Speaker =>
  def greet() = speak("Hi my name is " + person.firstName + " " + person.lastName)
}

//Mixin traits during instance creation
val person = new Person("Ted", "Steele") with Speaker with Greeter

person.greet() //Prints 'Hi my name is Ted Steele'

Function

Basics

val names = List("Adam", "Stellan", "Emma", "Li")

//Defines a anonymous function and stores it in value countLength
//the type String => Int is actally not needed as it can be inferred
val countLength: String => Int = (input: String) => input.length()

//Goes through the list and applies the function on each element
//creating a new list
val nameLength = for(name <- names) yield countLength(name)

//You can do the same using the collection method map in scala
val nameLength = names map countLength //Same as names.map(countLength)

//defines a method that does the same as the previously declared function
def countLength(input: String) = input.length()

//countLength method get converted to a function at compile time,
//if it is used as a function
val nameLength = names map countLength
